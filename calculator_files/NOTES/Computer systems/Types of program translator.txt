# Types of program translator

## Assembler

- translates ONLY assembly language into machine code
- due to the 1-to-1 relationship between assembly and machine code, translation is quick + straightforward
- assemblers are platform specific
  - different assembler must exist for each different type of processor instruction set

## Compiler

- translates high-level source code into machine code
- Translates the entire program at once
- Creates an executable file (unless there are any errors)
  - this can run without needing any other software to be present
- Provides error report of ALL the errors if there are any

## Interpreter

- translates high-level source code into machine code also
- Translates line-by-line
- Checks for errors as they go (ie line-by-line)
- Interpreter translates a program each time its executed
## Comparison Between Compiler and Interpreter

| Compiler | Interpreter |
| --- | --- |
| Analyses the program as a whole | Analyses program on a line-by-line basis |
| Produces object code / executable file / machine code / bytecode | Interpreter calls subroutines within its own code to carry out commands |
| Compiler will not translate any of the program if it encounters an error | Interpreter translates/executes program until first error is encountered |
| If (unchanged) program executed twice/multiple times, compiler will only need to translate it once | Interpreter translates a program each time it is executed |
| Once translated, compiled code does not need the compiler to be present to run | An interpreter must always be present for a program that is interpreted to run |
| Once compiled, code will only run on one type of processor / virtual machine | Interpreter could translate the same instruction multiple times (eg if it is in a loop) |
| If student has written about compiler outputting bytecode then: bytecode will later be interpreted // executed by a virtual machine // just-in-time-compiled | Interpreter executes each line immediately after translating it |

## Why Intermediate Languages Are Used?

- allows processor/platform independence
- compiler is only required to translate source code once
- can create intermediate language code for a specific (standardised) virtual machine that all the target machines have installed;
- VM can perform security checks on the (intermediate language / byte) code without execution, hence it is more secure;

## How Intermediate Languages Are Used?

- the code (now translated to the intermediate lang, eg bytecode) is executed by a virtual machine
- each different processor instruction set will have its own VM

- Source code = the input to a translator (ie what the programmer writes)
- Object code = the output from a translator

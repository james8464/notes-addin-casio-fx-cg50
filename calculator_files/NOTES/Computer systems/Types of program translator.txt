# Types of program translator

## Assembler

- translates only assembly language into machine code
- translation is quick + straightforward due to 1-to-1 relationship between assembly and machine code
- assemblers are platform specific
  - different assembler must exist for different types of processor instruction set

## Compiler

- translates high-level source code into machine code
- translates entire program at once
- only needs to translate the program once
- produces executable file (unless there are any errors)
  - can run without any other software dependencies
  - this can run without needing any other software to be present
  - can only run on one type of processor/VM
- provides error report of all errors

## Interpreter

- translates high-level source code into machine code
- executes each line immediately after translating it
- translates line-by-line
- checks for errors as they go (ie line-by-line)
- program is translated each time its executed




## Intermediate Languages (Bytecode)

### Why are Intermediate Languages Used?

- allows processor/platform independence
- improves portability (cross-platform execution)
- source code only needs to be compiled once
- same bytecode can run on multiple platforms
- compiler can generate code for a standardised virtual machine rather than a specific processor
- VM can perform security checks before execution
- more secure than directly executing native machine code
- each processor architecture only needs its own VM/JIT compiler rather than its own compiler

### How are Intermediate Languages Used?

- source code is compiled into an intermediate language (eg bytecode)
- bytecode is not machine code
- bytecode cannot be executed directly by the processor
- bytecode is executed by a virtual machine (VM)
- each processor architecture has its own VM
- VM translates/interprets bytecode for the specific processor
- VM may execute the bytecode directly through interpretation
- VM may use a JIT (Just-In-Time) compiler
- JIT compiler translates bytecode into native machine code at runtime
- JIT improves execution speed compared to pure interpretation

### Key Exam Comparisons

- machine code -> processor specific
- bytecode -> platform independent
- machine code -> executed directly by CPU
- bytecode -> requires VM/JIT
- compiler to machine code -> one executable per processor architecture
- compiler to bytecode -> same bytecode can run on many architectures

### Key Terms

- source code = code written by the programmer
- object code = output from a translator
- bytecode = intermediate language produced by some compilers
- virtual machine (VM) = software that executes/translates bytecode
- JIT compiler = converts bytecode into machine code during execution

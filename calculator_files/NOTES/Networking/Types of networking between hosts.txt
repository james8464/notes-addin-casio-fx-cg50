# Types of networking between hosts

## Client-server Networking

- one or more central servers provide services to clients
- clients request services from the server, which responds
- resources are stored on the server
- server provides services such as:
  - file storage
  - email management
  - user accounts
  - print queues

### Characteristics

- centralised security management
- centralised administration
- clients access resources from server
- server provides resources in response to requests

### Advantages

- centralised + more control over management of security + access rights
- easier administration (users, files, permissions)
- all data is stored centrally, and is accessible from any device
- centralised backups make it easier to backup data
- centralised/remote installation and updating of software is possible
- server can be optimised for specific tasks, eg file handling
- suitable for large number of users/devices

### Disadvantages

- configuration/setup can be more complicated and require expertise
- more expensive (server hardware required)
- dependency on the server introduced

### Where Used?

- Schools, businesses, organisations
- Where:
  - many users need managing
  - confidential data is stored
  - different users need different access rights
  - central control is required






## Peer to peer (p2p) Networking

- no central server
- resources are stored on each individual computer/device/peer
- each device can act as a client or a server
- devices communicate directly with each other

### Characteristics

- each computer has equal status
- any device can share/access resources with any other
- files can be distributed across computers
- No dependence on a server

### Advantages

- more cost effective as no server is needed
- easier to setup and maintain, not necessarily requiring expertise
- no single point of failure (no central server)
- useful for file sharing networks/multimedia sharing

### Disadvantages

- management of security/administration is more difficult
- no central control of access rights
- data is only available when the device storing it is switched on 
- if a device fails or is switched off, the resources stored on the device become unavailable to other devices connected to the P2P network
- no central backups
- not suitable for large networks

### Where Used?

- Small/home networks
- File sharing (e.g., media sharing)
- Where:
  - Low cost is important
  - Simple setup is needed
  - Few users

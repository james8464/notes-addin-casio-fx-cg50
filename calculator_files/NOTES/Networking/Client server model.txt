# Client server model

## Main Principle

- based on sender and receiver
- client initiates communication by sending a request
- server responds with a response message
- server provides the requested resource/service



## Servers

### Purpose

- provide services and resources to clients
- specialise in particular tasks

### Examples

- database servers
- file servers
- email servers




## Websocket Protocol

### Main Characteristics

- an example of an API
- operates in the application layer of the TCP/IP stack
- creates a persistent connection between client and server
- provides a constant stream of information between two devices
- usually connects a web browser and a server
- establishes a full-duplex communication channel
- data can be transmitted in both directions simultaneously
- allows ongoing communication without repeatedly refreshing a web page
- reduces overhead caused by repeated HTTP requests
- allows fast transmission of data by reducing packet-header overhead

### Why Used

- real-time communication
- continuous data exchange
- low latency communication

### Where Used

- video streaming
- online games
- instant messaging
- live updates and notifications






## CRUD

### Meaning

- acronym for:
    - create
    - retrieve
    - update
    - delete

### Purpose

- defines the operations that can be performed on data
- commonly used when interacting with databases

### CRUD → SQL Mapping

- create → insert
- retrieve → select
- update → update
- delete → delete







## REST

### Meaning

- representational state transfer

### Main Characteristics

- design methodology for online database applications
- used when databases are accessed through a web browser
- provides a standard way for systems to communicate over a network
- relies on HTTP request methods
- allows JavaScript to communicate with servers through HTTP
- REST APIs are created and run on the server
- browser JavaScript calls the REST API
- resources are represented by URLs
- entering a URL can cause the server to retrieve data from a database

### HTTP → SQL Mapping

- post → insert
- get → select
- put → update
- delete → delete

### Key Difference (Exam Point)

- crud = what operation you want to perform on data
- REST = how that operation is performed over a network






## JSON vs XML

### XML (Extensible Markup Language)

- markup language
- uses nested tags
- more verbose
- larger file size due to tags
- slower to parse
- more complex syntax
- very flexible through custom tags
- data typically stored as text
- commonly used in older systems and document storage
- example:
    - <name>Bob</name>

### JSON (JavaScript Object Notation)

- data-interchange format
- uses key-value pairs
- uses objects and arrays
- more compact
- easier for humans to read
- smaller file size
- faster to parse
- simpler syntax
- supports multiple data types
- commonly used in web APIs and modern applications
- example:
    - "name": "Bob"

### Advantages of JSON over XML

- easier for humans to read
- more compact
- easier to create
- easier for computers to parse
- quicker to parse
- typically requires less storage space
- facilitates faster transmission

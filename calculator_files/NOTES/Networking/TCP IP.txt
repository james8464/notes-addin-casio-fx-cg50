# TCP IP

- set of communication protocols used to connect hosts on a network
- allows computers on the same network to identify and communicate with each other
- TCP - responsible for end-to-end communication
- IP - responsible for routing packets from host to host


## The Layers

- application layer
  - selects and uses the correct protocol to transmit data
  - interacts with the user via application software (eg a browser)
- transport layer
  - establishes an end to end connection between the sender and receiver
  - splits the data into packets and adds header information (eg port number)
  - performs error detection
- network layer
  - adds source and IP addresses to packets
  - routers work within the network layer, so all functions of the router can be said to be functions of the network layer
    - eg performing routing, error detection on the packet header, etc
- link layer
  - controls physical connections between pieces of hardware in a network
  - adds MAC addresses to packets received from network layer
  
  

## The Process

- The file is passed down each layer in turn
- Application layer
  - Application software (eg file server) prepares the file
  - Selects protocol (eg FTP, SMB etc)
  - Data is passed to transport layer
- Transport layer
  - Establishes an end-to-end connection
  - Splits the file into packets
  - Adds:
    - Sequence numbers
    - Port numbers
  - Passes packets to the network layer
- Network layer
  - Adds:
    - Source IP address
    - Destination IP address
  - Determines routing across networks
  - Passes packets to the link layer
- Link layer
  - Adds:
    - Source MAC address
    - Destination MAC address
  - Sends data onto physical medium (eg cable)
- Receiving device - link layer
  - Receives data from medium
  - Removes MAC address
  - Passes packet up
- Network layer
  - Removes IP addresses
  - Passes packet up
- Transport layer
  - Uses port number to identify application
  - Uses sequence numbers to reorder packets
  - Reassembles data
- Application layer
  - Receives complete data
  - Reconstructs the file
  - Presents it to the user



## Socket

- Definition: a combination of an IP address and a port number

### Written as

  - IP address : port number
  - eg
  - 114.26.20.199:443 <- known as a 'socket address'

### Socket purpose

- Specifies which host (IP address) and which application/service on that host (port number) the data should be sent to

## How Sockets Are Used in Communication

### Sending Data

- Application chooses a port number
- Device has an IP address
- These combine to form a socket
- Socket is added to outgoing packets

### Receiving Data

- Packet arrives at device (using IP address)
- Transport layer reads port number
- Socket identifies which application should receive the data





## MAC address

- Definition: media access control

### MAC address purpose

- Identifies the hardware a packet should be sent to
- In routing, its changed with every hop through a network as it needs to be sent to a different router after each hop
- Unique to every hardware device
- The physical MAC address is on the NIC

## Well-known ports

- Definition: port numbers with a specific purpose, assigned by IANA
- FTP
  - file transfer protocol
  - port 20 and 21
  - used for sending files between devices
  - used for the connection between a FTP server and a client
- SSH
  - secure shell
  - port 22
  - used for remote management of devices
  - requires username + password
  - information is encrypted during transmission
- HTTP
  - hypertext transfer protocol
  - port 80
  - for loading webpages
- HTTPS
  - hypertext transfer protocol secure
  - port 443
  - same as HTTP, but with encryption
- POP3
  - post office protocol 3
  - ports 110 and 995
  - used for retrieving emails from an email server
- SMTP
  - simple mail transfer protocol
  - ports 25, 587, 465
  - used to send/transmit/receive emails

# Internet security

## Firewalls

- a network security device or software that acts as a barrier between a trusted internal network and an untrusted external network
- sits between a device and the Internet
- regulates the packets that pass through it
- can be either software or hardware

### Types

- packet filtering
  - packets are accepted/blocked
  - looks at the source IP address or the protocol they are using
  - network admin can specify particular IP addresses/protocols to block/allow
  - automatic filtering software can be used
- stateful inspection
  - contents of packet (ie payload) is examined before deciding whether to allow it
  - some firewalls keep a record of current connections in a network
    - this means it can filter out packets that aren't related to any activity on the network
  - can block/detect advanced threats, eg malware hidden in encrypted traffic
- proxy server
  - = a server that sits between a public network and a private network
  - manages every packet that passes between the two networks







## Symmetric Encryption

- relies on one key which can encrypt and decrypt the plaintext/ciphertext
- the sender and receiver share the same key

### Process

- before sending any information, the sender and receiver participate in a key exchange
  - ensures they both have a copied of the shared key
  - but vulnerable to interception 
- sender encrypts the plaintext into cipher text
- cipher text is sent to the receiver
- receiver decrypts the cipher text using the same key (step 1)






## Asymmetric Encryption

- overcomes the weakness of symmetric encryption
- 4 keys are used:
  - senders public key
  - senders private key
  - receivers public key
  - receivers private key
- the 2 keys each device has are mathematically linked
- as the names suggest, the private keys are kept private (so only that device knows it), and the public keys are public (ie anyone can see it on the Internet)
  - since the pairs are linked, it means a message encrypted with a public key can only be decrypted by the corresponding private key

### Process

#### At Sender Device

- sender applies hashing algorithm to plain text to make message digest
- message digest is encrypted with senders private key to make digital signature
- this is appended to the plain text
- this is encrypted with the receivers public key

#### At Receiver Device

- receivers private key used to decrypt ciphertext (leaves plaintext and digital signature)
- the plaintext is used to create a new message digest (using same hashing algorithm)
- senders public key is used to decrypt the digital signature to get the received message digest
- the two digests are compared
  - if they match, then it confirms that the sender sent the message and there were no errors in transmission




### Digital Certificates?

- it verifies ownership of a key pair used in asymmetric encryption
- can be used to check that a fake key pair isn't being used by an imposter
- issued by certificate authorities
- contains:
  - serial number
  - owner's name
  - expiry date
  - owner's public key
  - certificate authority's digital signature






## Worms

- can self-replicate and distribute independently
- does not need a host program
- does not actually harm any files
- usually affects storage + bandwidth

## Trojans

- program that disguises itself to be something the user wants
- requires the user to open/run it
- when installed, it can delete data + damage hardware
- often spread as email attachments or downloaded from bad websites

## Viruses

- a piece of code that replicates itself, spreading to other files
- can be used to damage the system/hardware
- requires a host file to reside (typically executable files)
- can spread between computers, private networks, the Internet, or even through physical media (eg dark drives)

## Spyware

- finds out information about the user and sends to hacker
- eg keyloggers

## Adware

- software that displays unwanted adverts across the system (without the user's consent)

## Ransomware

- encrypts user's files
- demands a ransom payment to decrypt the files
- usually demands payment through digital currencies (eg bitcoin) as it's a decentralised and anonymous platform





### Solutions to the Malware's Listed Above

- use anti-malware
  - provides real-time protection
  - when a file is opened/downloaded, it checks it against its known database of malwares
  - malware files are quarantined to prevent it from spreading, then deleted
  - the software dynamically updates the database (for all users) by making a signature of the malware and appending it to the database on the companies servers
  - the software can also check all files on the device for malware
- utilise encryption
  - secures data so it cant be understood by eavesdroppers/hackers
- use firewalls
  - data coming in/out of computer goes through firewall (set of rules)
  - if it does not match the rules, it is dropped
  - can look into packet contents and block certain ports
- use access levels
  - determines the permissions that each user/application has in regards to data on the device
  - eg you may block terminal from modifying content in your 'home' folder
- review privacy settings regularly
  - helps prevent the effectiveness of hacking attempts
  - eg turning off microphone/camera for certain apps, changing passwords regularly, 2fa, etc
- enable automatic updates of applications/OS
  - ensures any code vulnerably/improvements fixed by the developer are applied to the app
- review code quality
  - ensure code does not allow buffer overflow
  - ensure code prevents programs writing to memory locations not allocated to them
  - test software for security issues / vulnerabilities
  - carry out a code review so the code is independently checked by another programmer
  - use the latest versions of libraries

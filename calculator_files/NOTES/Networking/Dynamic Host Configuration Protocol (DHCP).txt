# Dynamic Host Configuration Protocol (DHCP)

## Purpose

- automates the configuration of hosts connecting to a TCP/IP network
- automatically allocates:
    - IP address
    - subnet mask
    - default gateway
- enables devices to join a network without manual configuration

## Why Used

- reduces need for expert knowledge when configuring a host
- reduces time required to configure hosts
- facilitates efficient use of a limited pool of IP addresses
- enables reuse of IP addresses when devices disconnect from the network
- helps avoid configuration errors
- prevents duplicate IP addresses being assigned
- reduces risk of incorrect network settings being entered manually

## Configuration Information Provided

- IP address
- subnet mask
- default gateway
- other network settings if required




## DHCP Process

### Step 1

- host sends a request to discover a DHCP server
- host requests network configuration

### Step 2

- DHCP server offers configuration information to the host
- offer usually includes an available IP address

### Step 3

- host accepts the offered configuration
- host echoes the accepted configuration back to the DHCP server

### Step 4

- DHCP server confirms the configuration has been allocated
- host can now communicate on the network using the assigned settings




## Why DHCP is Useful

- allows large numbers of devices to be configured automatically
- simplifies network administration
- makes adding new devices to a network easier
- reduces setup time




## Why its undesirable to let network settings of a web server to be defined by a DHCP server (PPQ)

- the IP address of the web server might be changed by the DHCP server
- port forwarding rules depend on the web server having a known IP address
- if the IP address changes, the router may no longer forward requests to the web server
- external users may be unable to access the website
- web servers typically require a fixed/static IP address

# The Internet and how it works

- = global network of interconnected computer networks
- uses TCP/IP for communication
- primarily a wired infrastructure (eg undersea cables connecting continents)

## ISP Hierarchy

- national ISPs provide backbone infrastructure
- regional/local ISPs connect homes and businesses
- shows Internet is hierarchical and decentralised




## Packet Switching

- data is split into packets (ie sections)
- packets sent independently
- each takes the best available route (fastest/shortest/most efficient)
- packets are reassembled at the destination according to their sequence numbers

## How routers are involved

- packets pass through multiple routers (hops)
- router:
  - reads destination IP
  - chooses next hop (not final destination) based on info in routing table
  - selects fastest/least congested route
  - decrements packet's TTL value (and deletes/kills it if it reaches 0)
- routing tables are updated dynamically
- each packet may take a different route, so they might arrive out of order
  - hence why there is a sequence number in the header

### Advantages

- efficient use of bandwidth
- fault tolerant (rerouting possible)
- supports multiple simultaneous transmissions

### Disadvantages

- packets can arrive out of order, requiring reassembly at the destination
- packets may be delayed due to network congestion
- packets can be dropped if their TTL expires or network issues occur
- missing packets must be retransmitted, increasing transmission time





## Packet

- packet = a container in which data is transmitted over networks
- contains a header, payload, and trailer

### Header

- sender's IP address
- receiver's IP address
- protocol identifier
- packet identifier
- sequence number
- number of packets in message
- TTL (time to live)
  - represents maximum number of hops for that packet
  - decreases by 1 each hop (ie at each router travelled to during the journey to the destination router)
  - if reaches 0 -> packet is dropped
  - purpose: prevents infinite looping
- packet size
- header length
- IP version

### Payload

- the actual data being transmitted in the packet

### Trailer

- end of packet signal
- error correction (eg checksum)
  - purpose: to check if the contents of the packet/data have been corrupted during transmission






## Define router

- device that connects networks together
- determines next hop using routing tables
- chooses best path (shortest/least congested)
- updates routing tables dynamically
- modifies MAC address for next hop
- drops packets when TTL expires

## Define gateway

- device that connects networks that use different protocols together
- removes packet header info, then rebuilds the packet to match the format the new protocol expects







## Uniform Resource Locator (URL)

- text-based address used to locate a resource on the Internet

### Components

- protocol - eg HTTP or HTTPS
- subdomain/host - eg www
- domain - eg bbc.co.uk
- path - eg /news/technology
- file name + extension - eg index.html

## Domain Names

- identifies an organisation or individual on the Internet
- uses alphanumeric characters, making it easier for humans to remember + type without errors
- maps to an IP address

- domain names are hierarchically organised
  - top-level domain (TLD)
    - eg .com
  - second-level domain (SLD)
    - eg .co
  - subdomains
    - eg
      - bbc
      - .co
      - .uk


## FQDN

- stands for: fully qualified domain name
- same as the domain name, but must include the server's host name (eg www.)
- eg: HTTPS://www.bbc.co.uk/news/index.html




## Domain Name Servers

- translates domain name into its corresponding IP address
- domain name servers store a table of domain names and their corresponding IP address
- if that server doesn't have a record of the domain your trying to access, the request it passed onto another DNS
- websites that are rarely visited will take longer to load since the request goes through more servers until the record is found


### Specific Functions

- name resolution
  - ie domain name to IP address (forward lookup)
  - also IP address to hostname (reverse lookup)
- organising the hierarchical structure of domain names
  - ie root, top level, 2LDs
  - managing name spaces


### How Domain Names Are Resolved

- the user types the FQDN into their browser
  - eg www.certbros.com
- check local cache first
  - the browser checks:
    - browser cache
    - operating system cache
  - if found:
    - IP address returned immediately
    - browser connects to website
  - if not found:
    - move to next step
- query recursive resolver (ISP)
  - a request is send to a DNS resolver (usually ISP)
- resolver checks its cache
  - if found:
    - returns IP address
  - if not found:
    - move to next step
- query root DNS server
  - this doesn't actually know IP addresses
  - it just tells the resolver where to find the TLD server (eg .com)
- query TLD server
  - this knows where the domain is managed
  - it returns the authoritative name server
- query authoritative DNS server
  - this stores actual domain records
  - returns the IP address
- return IP + cache it
  - resolver sends IP back to computer
  - IP is cached for future use
  - browser loads the website






## Internet Registries

### Purpose

- registers domains to people/organisations/companies
- allocates and manages IP addresses globally
- maintains records of domain ownership

### Why Needed?

- ensures domain names are unique
- needed to actually map a domain name to an IP address and store in the DNS system
- ensures efficient use of address space (eg reuses unused ips before issuing new ones)







## IP Address

- a unique numerical identifier for devices on a network
- can be ipv4 or ipv6
- not easy for humans to remember, which is why domain names are used
  - domain names map to IP addresses

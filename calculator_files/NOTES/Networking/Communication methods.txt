# Communication methods

## Serial Data Transmission

- data is sent one bit at a time over a single wire
- does not suffer from skew (bits arriving out of order)
- suitable for medium to long distances
- cheaper to manufacture/buy as less materials are needed
- cabling allows more flexibility over positioning
- slower than parallel as only one bit is sent simultaneously


## Parallel Data Transmission

- multiple bits sent simultaneously across multiple wires
- faster transmission rate than serial transmission
- more data can be transferred simultaneously
- used internally where high speed is required, eg the data bus
- data skew may occur (bits arriving out of sync)
- data corruption is more likely over longer distances
- more expensive
- less reliable over long distances





## Synchronous Data Transmission

- data transferred at regular intervals
- timed by a clock signal
- allows for constant and reliable transmission for time-sensitive data (eg real-time video or voice)
- the data sent will always be received in the same order that they were sent
- used within the busses of a computer's processor in the FE cycle



## Asynchronous Data Transmission

- no requirement for a shared clock signal
- receiver and transmitter (clock) do not need to be exactly synchronised to transmit data
- receiver and transmitter clock only synchronised at start of data transmission

### Purpose of start/stop bits

  - start bit = starts the receiver clock
  - stop bit = allows the next start bit to be recognised

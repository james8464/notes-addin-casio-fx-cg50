# Standard application layer protocols

- FTP
  - file transfer protocol
  - port 20 and 21
  - used for sending files between devices
  - used for the connection between a FTP server and a client
- SSH
  - secure shell
  - port 22
  - used for remote management of devices
  - requires username + password
  - information is encrypted during transmission
- HTTP
  - hypertext transfer protocol
  - port 80
  - for loading webpages
- HTTPS
  - hypertext transfer protocol secure
  - port 443
  - same as HTTP, but with encryption
- pop3
  - post office protocol 3
  - ports 110 and 995
  - used for retrieving emails from an email server
- SMTP
  - simple mail transfer protocol
  - ports 25, 587, 465
  - used to send/transmit/receive emails
- IMAP
  - Internet message access protocol
  - port 143
  - unencrypted connections


## FTP client software and FTP servers

- FTP = method used for transferring files/data across a network, often the Internet
- an application layer protocol
- files can be dragged from the folder structure of the remote site or server
- FTP sites may also be used by software companies offering large updates, or by press photographers to upload their latest photographs to a remote newspaper headquarters.
- most FTP sites require a username and password to authenticate the user, but some sites could be configured to allow anonymous use without the need to login.


# SSH used for remote management

- = an encrypted protocol that allows secure communication between nodes across a network

## SSH Clients

- pieces of software
- used to make a TCP connection to a remote port (via port 22)
- this connection allows commands for application-level protocols to be sent to the remote server






### Role of a Web Server

- stores web pages (HTML files)
- receives requests for those pages
- responds to the requests by sending pages as text using HTTP/HTTPS

### Role of a Web Browser

- a web browser sends requests to a web server asking for web pages
- it then receives the HTML text and all other resources (images, css, scripts) needed for the page
- browser downloads these files, interprets them, and renders the finished web page to the user

# Network topology

## Physical vs Logical Topology

### Physical Topology

- the physical layout of the cabling/wiring/connections
- shows how devices are actually connected (cables, hardware)

### Logical Topology

- how data flows around/through the network
- not necessarily the same as the physical layout
- describes the path data takes

### Key Idea

- Physical = what it looks like
- Logical = how it works

## Topology Structures

### Star Topology

- all devices connect to a central device (e.g., switch/hub)

### Advantages

- if one cable/device fails, the rest of the network is unaffected
- easy to add/remove devices/nodes
- easy to identify faults
- good performance (dedicated connections)

### Disadvantages

- if central device fails, entire network fails
- requires more cabling (regarding the physical topology)
- more expensive

### Bus Topology

- all devices connected to a single main cable (backbone)

### Advantages

- uses less cable -> cheaper
- simple to setup

### Disadvantages

- if backbone fails, whole network fails
- data collisions can occur
- performance slows with more devices
- harder to troubleshoot faults

### Ring Topology (beyond Spec)

- devices are connected in a closed loop
- data travels in one direction

### Advantages

- no collisions (data flows in one direction)
- predictable performance

### Disadvantages

- if one device/cable fails -> can break network
- harder to add/remove devices

# Data compression

## Why?

- reduces storage requirements of files
- faster transmission times
- lowering cloud storage costs
- enhances network bandwidth efficiency
- faster backups
- email attachments (reducing document size to stay within attachment limits)
- images and sound files are often compressed because they can be very large
- text files can also be compressed
- compression means fewer bits are needed to represent the same data





## Lossy Compression

- some information is lost in the process
- original data cannot be recovered if lossy compression has been used
- removes data judged to be unnecessary or less noticeable
- usually gives a smaller file size than lossless compression
- often used for images, sound and video
- not suitable when the original data must be recovered exactly

### Advantages

- can greatly reduce file size
- faster to transmit than the original file
- useful for streaming and online media

### Disadvantages

- data is permanently lost
- quality may be reduced
- repeated lossy compression can reduce quality further





## Lossless Compression

- no loss of information
- size of a file is reduced without decreasing its quality
- original data can be fully recovered if lossless compression was used
- works by removing redundancy
- suitable when the exact original data is needed
- used for text, programs, documents and some image formats

### Advantages

- original file can be reconstructed exactly
- no reduction in quality
- suitable for important data where every bit matters

### Disadvantages

- usually gives a smaller reduction in file size than lossy compression
- may not compress files much if there is little repetition/redundancy





## Run length Encoding (RLE)

  - = identifies repeated information, and replaces it with one occurrence of the repeated information, followed by the number of times it is to be repeated
  - a run is a sequence/series of a particular piece of data
      - eg in an image, there might be a run of pixels that are the same
  - a run stores:
      - the data value
      - the number of times it repeats
  - best when there are long runs of repeated data
  - commonly useful for simple bitmap images with large areas of the same colour
  - less effective when data changes frequently
  - can increase file size if there are very few repeated runs
  - lossless compression technique





## Dictionary-based Methods

- repeated patterns of data are identified + stored in a dictionary
- repeated data is replaced with the key to that data in the dictionary
- not as effective for files with little repetition
- the dictionary stores repeated words, symbols or sequences
- repeated data is replaced with a shorter token/reference
- dictionary must be available during decompression
- works well when the same patterns appear many times
- less effective when data has few repeated patterns
- lossless compression technique

### Process

- scan the data for repeated patterns
- add repeated patterns to a dictionary
- replace later occurrences with shorter references
- use the dictionary to rebuild the original data during decompression

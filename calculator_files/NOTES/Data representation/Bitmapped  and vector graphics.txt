# Bitmapped graphics and vector graphics

# Bitmapped graphics

- images made from pixels
- pixel = smallest addressable picture element. contains one colour
- resolution = num of pixels used to make the bitmap, measured in dots per inch
- colour depth = num of bits allocated to represent the colour of each pixel
- very realistic images
- high detail and subtle colour variation
- easy to edit at pixel level
- no processing needed to render, just display pixels
- larger file sizes
- pixelation when enlarged
- quality depends on resolution
- not easily scalable
- editing large images can be slow

## Uses

- real world photography
- photos of people


## Formula to Calculate File Size (of Bitmap Image)

  - file size = resolution * colour depth
  - however, this value is a minimum. in reality, its higher due to metadata



# Vector graphics

- represents images using geometric objects
- properties are stored in a list for each object
- can be scaled without loss of quality
- smaller file sizes as only data about the objects need to be stored rather than each pixel
- easy to edit objects
- does not result in a void area if an object is deleted
- cannot represent fine detail or complex textures well
- more processing required to render
- limited colour realism compared to bitmaps


## Uses

- CAD packages
- 2d/3d programs
- chart, logo, map, plan, clipart
- SVG images

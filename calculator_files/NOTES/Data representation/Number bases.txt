# Number bases

## Binary

- binary is base 2
- directly understood/processed by computers
- simple and reliable hardware implementation
- no conversion needed for computer processing
- useful for representing individual bits and logical values
- difficult for humans to read and write
- long strings of digits are needed for large values
- more prone to human transcription errors
- harder for humans to perform manual calculations
- less compact than hexadecimal



## Decimal/denary

- decimal/denary is base 10
- uses the digits 0-9
- standard number base used by humans
- easier for humans to understand and remember
- convenient for everyday counting and calculations
- easier for humans to perform arithmetic manually
- used globally, allowing values to be compared easily
- not directly understood by computers
- must be converted to binary for processing
- less suitable for representing individual bits
- does not map directly to hardware states
- less compact than hexadecimal for displaying large binary values



## Hexadecimal

- hexadecimal is base 16
- uses the digits/symbols:
- 1 hexadecimal digit represents 4 bits
- direct conversion to/from binary is simple
- much shorter than binary
- requires fewer characters and less screen space
- makes large binary values easier to display and remember
- humans are less likely to make mistakes when copying values
- computers do not process hexadecimal directly
- mainly used for human convenience
- less intuitive than decimal for beginners
- letters a-f can be confusing
- not as familiar to most users as decimal

## Uses of Hexadecimal

- memory dumps
- MAC addresses
- HTML colour codes
- machine code/debugging displays
- representing long binary values more compactly

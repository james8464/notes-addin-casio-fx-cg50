# Encryption

- definition: the encoding of a message so that the message can only be decrypted and read by the authorised receiver(s)
- scrambles the data so it cannot be understood if intercepted
- keeps data secure during transmission
- plaintext = the unencrypted message/data
- ciphertext = the encrypted message/data
- cipher = a type of encryption method

## Computationally Secure

- a cipher is computationally secure if:
  - its theoretically breakable but not in a useful timeframe using current technology
- the cipher cannot be cracked in a reasonable amount of time

## Caesar Cipher

- encrypts info by replacing characters
- one character is always replaced by the same character (weakness)
- there are two types:
  - shift ciphers - each letter in the alphabet is shifted by the same amount
- substitution ciphers - letters are randomly replaced

### Weaknesses?

- subject to frequency analysis
  - the frequency at which each character occurs can provide clues as to which letter has been replaced with which
- ciphertext retains structural properties of the plaintext message
  - eg some letters will still frequently appear next to each other
  - spaces can be used to analyse word lengths

## Vernam Cipher

- each key should only ever be used once
- key must be truly random and at least as long as the plaintext that is to be encrypted

### How Does It Work?

- aligns the characters of the plaintext and the key
- converts each character to binary
- applies XOR operation to the two bit patterns
- converts result back to characters


### Perfect Security?

- only cipher mathematically proven to be completely secure
- since the keys are random, the ciphertext is random, so cipher is completely secure

  - all ciphers (other than vernam cipher) are, in theory, crackable, but not in a reasonable/useful timeframe given current computing power
  - ciphers that use this form of security are said to rely on computational security

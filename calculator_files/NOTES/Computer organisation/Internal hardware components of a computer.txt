# Internal hardware components of a computer

## Components

- processor
  - carries out computation on data by following instructions
  - handles all of the instructions received from user + hardware + software
- main memory
  - stores data/instructions that are currently or will be used by the processor
  - returns the instructions/data/value stored in a memory location
  - stores any value/data resulting from the execution of a program
- address bus
  - uni-directional
  - carries address signals from CPU to main memory and I/O devices
- data bus
  - bi-directional
  - carries data and instructions between memory, I/O controllers, and the CPU
- control bus
  - bi-directional
  - transmits command, timing and specific status information between system components
- I/O controllers
  - located between the processor and the I/O device
  - controls the flow of data between the processor and I/O



## Roles of an I/O Controller

- allows processor to control/communicate with a peripheral using an I/O port
- allows peripheral to appear as a set of memory locations to the processor
- translates signals/data from the peripheral into a form that the processor can understand
- allows new peripherals to be added without having to redesign the processor/hardware
- checks if data received from peripheral is corrupted or not
- generates an interrupt when data is ready to be transferred from an I/O device





# Von Neumann vs Harvard Architecture

## Von Neumann

- data and instructions are stored in the same memory
- data and instructions share the same address space
- the same buses are used to fetch instructions and transfer data
- processor cannot fetch an instruction and access data at the same time
- shared bus system can cause the Von Neumann bottleneck
- bottleneck = CPU waits for memory access because instructions/data must use the same route
- simpler and more flexible than Harvard architecture
- used in everyday general-purpose computer systems (eg laptops + smartphones)
- stored program concept = instructions are stored in main memory with data, using discrete unique addresses
- program instructions can be loaded, stored and treated like data

## Harvard Architecture

- data and instructions are stored in separate memories
- data and instructions have separate address spaces
- separate buses are used for instruction fetches and data transfers
- instructions and data can be accessed at the same time
- reduces the Von Neumann bottleneck
- different word lengths can be used for instructions and data
- different memory sizes and memory technologies can be used for instructions and data
- instruction memory may use ROM/flash so the program cannot be easily changed
- harder for data to accidentally overwrite program instructions
- more complex and less flexible than Von Neumann architecture
- mainly used in embedded systems and microcontrollers


## PPQ: Reasons to Use Harvard Rather Than Von Neumann

- instructions and data can be accessed simultaneously
- reduces bottleneck caused by shared data/address buses
- reduces delays waiting for memory fetches
- avoids possibility of data being executed as code (which is one method that can be exploited by hackers)
- being able to use exclusively ROM for instruction memory prevents the program being modified/hacked
  - program cannot be accidentally overwritten by data
- instruction and data memory can have different word lengths
- different technologies can be used to implement instruction and data memory
- different quantities of instruction and data memory means that address lengths can differ between the two
- memory address structures can differ

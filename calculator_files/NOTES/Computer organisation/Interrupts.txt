# Interrupts

- a signal sent to the processor from a hardware device or program
- examples:
  - moving a mouse
  - pressing a keyboard key
  - printer paper jams
  - division by zero
  - a printer finishing its job
  - stack overflow

## Purpose of interrupts

- allows the currently executing process to be suspended
- so that a device/source that needs immediate attention of the processor can be serviced
- allows computers to multitask

## Interrupt Service Routine (isr)

- = software routine the processor invokes in response to an interrupt
- designed to process each interrupt
- origin of the interrupt is identified so that the appropriate ISR is called
- all other lower-priority interrupts are put on hold to allow the ISR to finish running
- the PC is updated with the address of the first instruction of the ISR
- the program counter is updated with the address of the first instruction of the ISR
- the ISR completes its execution
- the processor registers are reloaded with the values that were saved to memory
- the lower-priority interrupts that were put on hold are re-established

### Note

- when the ISR is processing interrupts, the values of registers may change
- this could cause data loss
- to prevent this, the data is saved (pushed on top of the system stack) before the interrupt is processed.

# Factors affecting processor performance

## Multiple Cores

- more cores = more instructions can be executed simultaneously
- each core can perform its own FE cycle independently of others

## Cache Memory

- cache = small portion of incredibly fast memory, much faster than HDD's/SSD's
- way faster than HDD's and even SSD's
- stores frequently accessed instructions/data
- data can be accessed more quickly -> programs run faster
- very expensive
- cache is v expensive

## Clock Speed

- relates to the frequency of clock pulses (generated by system clock)
- these pulses synchronise the computer
- higher clock speed -> more FE cycles carried out per second -> faster processing of tasks
- higher clock speed -> higher power consumption (overheating? -> reduce performance)

## Word Length

- word = a group of bits that is treated as a single unit by a processor
  - length of a word = number of bits assigned to it
- higher word length -> more bits can be transferred as a single unit
      - = more data can be moved in a single operation
      - = speeds up tasks involving large data transfers

## Address Bus Width

- determines the number of unique memory addresses the processor can access
- width of a bus relates to the num of parallel wires that make up the bus
- eg a 32-bit address bus can address 2^32 unique memory locations
- more unique memory locations -> reduces need for secondary storage / virtual memory
- more unique memory locations -> reduces need for secondary storage / virtual memory (these are slower)

## Data Bus Width

- larger data bus width = higher volume of data can be transferred over the bus at any one time
- eg a 32-bit data bus can transfer 32 bits at once
- wider data bus -> less FE cycles needed to fetch instructions from memory -> faster executions

## Other Factors

- use parallel processor architecture / harvard architecture
- use processor that makes use of pipelining
- install more RAM
- use RAM with faster access time
- use SSD rather than HDD for secondary storage

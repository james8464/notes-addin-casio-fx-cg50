# Functional programming paradigm

## Function type

- If f: A -> B, the function type is A -> B.
- A is the argument type.
- B is the result type.
- Function = rule that for each element in set A of inputs, assigns an output chosen from set B.
- Not all elements of set B will be used.
- A function maps a set of inputs, called the domain, to a set of possible outputs, called the co-domain.
- Functions only calculate something and return a result.
- Functions have no side effects, so they cannot change the data / memory.
- Every time the same function is called with the same arguments / values, it will always return the same result.
- Domain = set that a function takes input values from.
- Co-domain = set that function output values are from.
- Types are integer / float / bool / char values.
- Typeclasses are sets of types, such as Integer, Float, Bool, Char.

### Type declaration

- Can be implicit - set by the programming language in the background.
- Can be explicit - set by the programmer.

### Explicit type declaration example

    isEqual :: Int -> Int -> Bool
    isEqual x y = x == y

## First-class object

- First-class objects, or values, are objects that may:
  - Appear in expressions
  - Be assigned to a variable
  - Be assigned as arguments
  - Be returned in function calls
- Note: the object term here is not related to OOP.
- Examples: integers, floating point values, characters, strings, functions.

## Function application

- Function application = a function applied to its arguments.
- Giving particular inputs to a function is function application.
- Example: add(3,4) is the application of function add to integer arguments 3 and 4.
- The type of this function is f: integer x integer -> integer.
- integer x integer is the cartesian product of set Integer with itself.

## Partial function application

- In the add example function above, it takes two integers as arguments and returns an integer as a result.
- As a partial function application, this is viewed as:

    add : integer -> (integer -> integer)

- This means add 4 returns a function which, when applied to another integer, adds 4 to that integer.
- This is why add function type is integer -> integer -> integer.
- It takes one argument after another and returns a result of data type integer.

## Composition of functions

- Functional composition combines two functions to get a new function.
- If f : A -> B, and g : B -> C, then g o f is called the composition of g and f.
- g o f is a function whose domain is A and co-domain is C.
- Composition is possible because the co-domain of f is the same as the domain of g.

## Higher order functions

- Higher order function = a function that takes a function as an argument.
- Higher order function = a function that returns a function as a result.
- Higher order function = a function that takes a function as an argument and returns a function as a result.
- A. Parameter / input for argument.
- NE. A function that uses another function.
- R. Explanations that are specifically of the map function.

### Higher order function example

    applyTwice :: (a -> a) -> a -> a
    applyTwice f x = f( f x)

- Parentheses are required and define the type declaration of function f.
- 'a' denotes a type - they have to be the same.

### applyTwice examples

    applyTwice (+3) 10

- Returns 16.

    applyTwice (++ " HAHA") "HEY"

- Returns "HEY HAHA HAHA".

    applyTwice ("HAHA " ++) "HEY"

- Returns "HAHA HAHA HEY".

    applyTwice (multThree 2 2) 9

- Returns 144, with multThree previously defined to multiply three input values.

    applyTwice (3: ) [1]

- Returns [3,3,1].

## Map, filter and reduce

- map is the name of a higher-order function that applies a given function to each element of a list, returning a list of results.
- filter is the name of a higher-order function that processes a data structure, typically a list, in some order to produce a new data structure containing exactly those elements of the original data structure that match a given condition.
- reduce or fold is the name of a higher-order function which takes a list of values and a function as arguments.
- reduce or fold repeatedly applies, using recursion, that combining function to each element of the list.
- reduce or fold returns a single value.

## Lists and immutability

- List = a collection of elements of a similar type, such as integers, characters or strings, enclosed in square brackets.
- Head = first element of the list.
- Tail = remainder of the list if you removed the head.
- A list can be written as head : tail.
- This is the concatenation of head and tail.
- Empty list = a list with no elements; [].
- length(listA) = number of elements in listA.

### List example

- Say that listA = [3,4,5,6].

    head(listA)

- returns 3.

    tail(listA)

- returns [4,5,6].

    tail(tail((tail(tail(listA))))

- returns [].

    null(listA)

- returns False.

    B = []

- defines the empty list B.

### Pre-pending a list

- Pre-pending a list = adding element(s) to the front of the list.
- The operator is : for a single element.

    5 : [1,2,3]

- returns [5,1,2,3].
- The operator is ++ for a list.

    [5,4] ++ [1,2,3]

- returns [5,4,1,2,3].

### Appending a list

- Appending a list = adding element(s) to the end of the list.
- The operator is ++ as you can only append a list to the existing list.
- Remember the original list will NOT change its contents.

    A = [1,2,3]
    A ++ [4,5]

- returns [1,2,3,4,5].
- The contents of list A remain the same.
- This is the concept of immutability.
- Concatenating lists also uses the ++ operator.
